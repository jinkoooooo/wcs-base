# ================ Commit Message Template ================
# 구조: <type>(<optional scope>): <description>
#
# type (소문자):
#   feat     : 새 기능 추가 (ex. 토탈피킹 기능, AMR 연동)
#   fix      : 버그 수정 (ex. UI 렌더링 오류 해결)
#   docs     : 문서 변경 (ex. README, 설치 가이드)
#   style    : 기능 영향 없는 스타일 변경 (ex. 세미콜론, 주석, import 정리)
#   refactor : 성능 변화 없이 코드 구조 개선 (ex. 서비스 클래스 분리)
#   perf     : 성능 개선을 위한 변경 (ex. 알고리즘 최적화)
#   test     : 테스트 코드 추가/수정 (ex. 단위 테스트, 더미 데이터)
#   build    : 빌드/환경 설정 변경 (ex. gradle 설정, 의존성 추가)
#   chore    : 환경설정 및 기타 (ex. .gitignore, IDE 설정)
#
# scope (소문자, 선택):
#   logic    : 내부 로직 (ex. WCS↔ECS, 화면 조회)
#   if       : 외부 시스템 연동 (ex. WMS, AMR, PLC)
#   common   : 공통 모듈/유틸 (ex. CommonPage.vue)
#   ui       : 화면/프론트엔드 (ex. TABLET, PDA)
#   db       : 데이터베이스 관련 (ex. 스키마 수정, 테이블 변경)
#
# description:
#   - 명령문 형태 (~추가, ~수정, ~삭제, ~개선 등)
#   - 마침표(.) 금지
#
# body (선택):
#   - 무엇, 왜 중심으로 상세히 작성
#   - description과 한 줄 공백으로 구분
#
# 예시:
#   feat(logic): 토탈피킹 기능 추가
#
#   토탈피킹 기능을 새로 구현하여 PDA 화면과 연동
